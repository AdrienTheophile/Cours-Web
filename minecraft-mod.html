🔧 Créer son Mod Minecraft avec Forge

Ajoute des items, des armes et des armures dans Minecraft en Java !

🏠 Accueil ✨ Débutant 🧩 Intermédiaire 🚀 Confirmé 📖 Langages 🎮 Roblox ⛏️ Minecraft

🎯 Objectif du cours

Dans ce tuto, tu vas apprendre à créer un mod Minecraft Forge qui ajoute :

  • 🧶 Un nouvel item — Le Coton (ingrédient)
  • ⚔️ Des armes personnalisées — Un sabre laser bleu et un rouge !
  • 🛡️ Une armure complète — La tenue de Jedi (casque, plastron, jambières, bottes)
⚠️ Attention : Les codes ci-dessous ne sont pas copiables ! Tu dois les recopier à la main pour apprendre et comprendre chaque ligne.

📋 Les étapes pour créer ton mod

  1. Installe le JDK 17+ — Tu as besoin de Java Development Kit pour compiler le mod.
  2. Télécharge le MDK Forge pour 1.21 — C'est le kit de développement de mods sur le site de Forge.
  3. Extrais le MDK et ouvre le projet dans IntelliJ IDEA (Community Edition gratuit).
  4. Attends que Gradle se synchronise — Ça peut prendre quelques minutes la première fois.
  5. Crée les fichiers Java — Recopie les codes ci-dessous dans les bons fichiers.
  6. Crée les fichiers JSON — Pour les modèles d'items et les traductions.
  7. Ajoute les textures — Dessine tes textures 16×16 pixels avec Paint.NET.
  8. Lance le jeu avec le bouton ▶️ dans IntelliJ pour tester ton mod !

📄 TestMod.java 🚫 À recopier

📍 Chemin : src/main/java/com/example/examplemod/TestMod.java

C'est le fichier principal du mod. Il enregistre tous tes items et les ajoute dans les onglets créatifs.

package com.example.examplemod; import com.example.examplemod.item.ModItems; import com.mojang.logging.LogUtils; import net.minecraft.world.item.CreativeModeTabs; import net.minecraftforge.api.distmarker.Dist; import net.minecraftforge.common.MinecraftForge; import net.minecraftforge.event.BuildCreativeModeTabContentsEvent; import net.minecraftforge.event.server.ServerStartingEvent; import net.minecraftforge.eventbus.api.IEventBus; import net.minecraftforge.eventbus.api.SubscribeEvent; import net.minecraftforge.fml.ModLoadingContext; import net.minecraftforge.fml.common.Mod; import net.minecraftforge.fml.config.ModConfig; import net.minecraftforge.fml.event.lifecycle.FMLClientSetupEvent; import net.minecraftforge.fml.event.lifecycle.FMLCommonSetupEvent; import net.minecraftforge.fml.javafmlmod.FMLJavaModLoadingContext; import org.slf4j.Logger; // The value here should match an entry in the META-INF/mods.toml file @Mod(TestMod.MOD_ID) public class TestMod { // Define mod id in a common place for everything to reference public static final String MOD_ID = "testmod"; public static final Logger LOGGER = LogUtils.getLogger(); public TestMod() { IEventBus modEventBus = FMLJavaModLoadingContext.get().getModEventBus(); modEventBus.addListener(this::commonSetup); // Register ourselves for server and other game events we are interested in MinecraftForge.EVENT_BUS.register(this); ModItems.register(modEventBus); // Register the item to a creative tab modEventBus.addListener(this::addCreative); // Register our mod's ForgeConfigSpec so that Forge can create and load the config file for us ModLoadingContext.get().registerConfig(ModConfig.Type.COMMON, Config.SPEC); } private void commonSetup(final FMLCommonSetupEvent event) { } // Add the example block item to the building blocks tab private void addCreative(BuildCreativeModeTabContentsEvent event) { if(event.getTabKey() == CreativeModeTabs.COMBAT) { event.accept(ModItems.LIGHTSABER_BLUE); event.accept(ModItems.LIGHTSABER_RED); } if(event.getTabKey() == CreativeModeTabs.INGREDIENTS) { event.accept(ModItems.COTON); } if(event.getTabKey() == CreativeModeTabs.COMBAT) { event.accept(ModItems.JEDI_HELMET); event.accept(ModItems.JEDI_CHESTPLATE); event.accept(ModItems.JEDI_LEGGINGS); event.accept(ModItems.JEDI_BOOTS); } } // You can use SubscribeEvent and let the Event Bus discover methods to call @SubscribeEvent public void onServerStarting(ServerStartingEvent event) { } // You can use EventBusSubscriber to automatically register all static methods in the class annotated with @SubscribeEvent @Mod.EventBusSubscriber(modid = MOD_ID, bus = Mod.EventBusSubscriber.Bus.MOD, value = Dist.CLIENT) public static class ClientModEvents { @SubscribeEvent public static void onClientSetup(FMLClientSetupEvent event) { } } }

📄 ModItems.java 🚫 À recopier

📍 Chemin : src/main/java/com/example/examplemod/item/ModItems.java

Ce fichier enregistre tous les items du mod : le coton, les sabres laser et l'armure Jedi.

package com.example.examplemod.item; import com.example.examplemod.TestMod; import net.minecraft.world.item.ArmorItem; import net.minecraft.world.item.Item; import net.minecraft.world.item.SwordItem; import net.minecraftforge.eventbus.api.IEventBus; import net.minecraftforge.registries.DeferredRegister; import net.minecraftforge.registries.ForgeRegistries; import net.minecraftforge.registries.RegistryObject; public class ModItems { public static final DeferredRegister<Item> ITEMS = DeferredRegister.create(ForgeRegistries.ITEMS, TestMod.MOD_ID); // ITEMS public static final RegistryObject<Item> COTON = ITEMS.register("coton", () -> new Item(new Item.Properties())); // ARMES public static final RegistryObject<Item> LIGHTSABER_BLUE = ITEMS.register("lightsaber_blue", () -> new SwordItem(ModToolTiers.LIGHT, new Item.Properties() .attributes( SwordItem.createAttributes( ModToolTiers.LIGHT, 5, -2f)))); public static final RegistryObject<Item> LIGHTSABER_RED = ITEMS.register("lightsaber_red", () -> new SwordItem(ModToolTiers.LIGHT, new Item.Properties() .attributes( SwordItem.createAttributes( ModToolTiers.LIGHT, 17, -2f)))); // ARMORS public static final RegistryObject<Item> JEDI_HELMET = ITEMS.register("jedi_helmet", () -> new ArmorItem(ModArmorMaterials.JEDI_COAT, ArmorItem.Type.HELMET, new Item.Properties().durability(ArmorItem.Type.HELMET.getDurability(18)))); public static final RegistryObject<Item> JEDI_CHESTPLATE = ITEMS.register("jedi_chestplate", () -> new ArmorItem(ModArmorMaterials.JEDI_COAT, ArmorItem.Type.CHESTPLATE, new Item.Properties().durability(ArmorItem.Type.CHESTPLATE.getDurability(18)))); public static final RegistryObject<Item> JEDI_LEGGINGS = ITEMS.register("jedi_leggings", () -> new ArmorItem(ModArmorMaterials.JEDI_COAT, ArmorItem.Type.LEGGINGS, new Item.Properties().durability(ArmorItem.Type.LEGGINGS.getDurability(18)))); public static final RegistryObject<Item> JEDI_BOOTS = ITEMS.register("jedi_boots", () -> new ArmorItem(ModArmorMaterials.JEDI_COAT, ArmorItem.Type.BOOTS, new Item.Properties().durability(ArmorItem.Type.BOOTS.getDurability(18)))); public static void register(IEventBus eventBus) { ITEMS.register(eventBus); } }

📄 ModToolTiers.java 🚫 À recopier

📍 Chemin : src/main/java/com/example/examplemod/item/ModToolTiers.java

Ce fichier définit le tier (niveau) de tes armes : durabilité, dégâts, vitesse de minage, etc.

package com.example.examplemod.item; import net.minecraft.tags.BlockTags; import net.minecraft.world.item.Tier; import net.minecraft.world.item.crafting.Ingredient; import net.minecraftforge.common.ForgeTier; public class ModToolTiers { public static final Tier LIGHT = new ForgeTier( 1400, // durabilité 1f, // vitesse de minage 14f, // bonus de dégâts 22, // enchantabilité BlockTags.MINEABLE_WITH_PICKAXE, // tag de blocs minables () -> Ingredient.EMPTY, // réparation BlockTags.INCORRECT_FOR_WOODEN_TOOL // blocs incorrects ); }

📦 Fichiers JSON à recopier

📄 coton.json 🚫 À recopier

Ce fichier JSON indique à Minecraft quel modèle utiliser pour afficher l'item Coton. Tu dois aussi créer un fichier similaire pour lightsaber_blue.json et lightsaber_red.json en changeant le nom de la texture.

{ "parent": "minecraft:item/generated", "textures": { "layer0": "testmod:item/coton" } }

📄 en_us.json 🚫 À recopier

📍 Chemin : src/main/resources/assets/testmod/lang/en_us.json

Ce fichier donne les noms affichés de tes items en jeu. Sans ce fichier, tes items apparaîtront avec des noms techniques illisibles.

{ "item.testmod.coton": "Coton", "item.testmod.lightsaber_blue": "Blue LightSaber", "item.testmod.lightsaber_red": "Red LightSaber" }

💡 Conseils et astuces

  • Textures : Les textures d'items doivent être des images .png en 16×16 pixels. Place-les dans assets/testmod/textures/item/.
  • Noms de fichiers : Utilise uniquement des minuscules, des underscores (_) et pas d'espaces ni d'accents dans les noms.
  • Modèles JSON : Chaque item a besoin d'un fichier .json dans models/item/ avec exactement le même nom que l'item enregistré dans le code.
  • Tester : Lance le jeu avec le bouton ▶️ dans IntelliJ. En jeu, ouvre l'inventaire créatif et cherche dans l'onglet Combat ou Ingrédients.
  • Erreurs : Si le jeu crash, lis le message d'erreur dans la console ! Souvent c'est un nom de fichier mal écrit ou un fichier au mauvais endroit.
💡 Pour aller plus loin : Tu peux modifier les valeurs dans ModToolTiers.java pour changer les dégâts, la durabilité ou la vitesse de tes armes !
